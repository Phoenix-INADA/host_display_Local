# PICO2 間 シリアル通信 インターフェース

概要
パソコンとPICO2基板（以下PICO2）をUSBで接続し、仮想COMポートを介して非同期テキストメッセージを交換するプロトコルを定義する。

共通ルール
- エンコーディング: UTF-8（ASCIIサブセットのみを使用）。
- フレーミング: 各メッセージはLF ("\n") で終端する。受信は行単位で処理すること。
- 区切り: フィールドはコロン ":" で区切る。フィールド間に空白文字は入れない。
- 最大長: 全メッセージ（終端含む）は 64 バイトを超えないこと（実装上の保護）。
- 不正メッセージ: 構文エラーや未定義コマンドは [PI]:ERR:<code>:<msg> （PI→PC）で通知する。

データ順序（重要）
- LEDバイト順: 緑1, 緑2, 緑3, 緑4, 赤1, 赤2, 赤3, 赤4 の順で8文字。インデックスは 0 から 7 として扱う。
- Buttonバイト順: BTN0〜BTN9 の順で10文字。
  - BTN0〜BTN3: 商品選択（商品0〜3）
  - BTN4: 10円投入
  - BTN5: 50円投入
  - BTN6: 100円投入
  - BTN7: 500円投入
  - BTN8: 金額クリア
  - BTN9: 商品補充

パソコン -> PICO2（コマンド）
1) LED 状態変更

1a) LED 個別指定（SET） — 単一LED指定（4chars）
- フォーマット: [PC]:LED:SET:<id>:<ctrl>\n
- <id>: デバイス識別子（GRN0..GRN3, RED0..RED3, BTN0..BTN9）
- <ctrl>: 許容値 '0'（消灯）, '1'（点灯）, '2'（点滅）
- 説明: 単一のLEDを指定して状態を設定する（例: [PC]:LED:SET:GRN1:2\n）。

1b) LED 一括指定 — 8文字ペイロード（8chars）
- フォーマット: [PC]:LED:<8chars>\n
- 8chars: 各LEDにつき1文字。並び順は 緑1, 緑2, 緑3, 緑4, 赤1, 赤2, 赤3, 赤4（合計8文字）。
- 許容値:
  - '-' : 何もしない（既存状態維持）
  - '0' : 消灯
  - '1' : 点灯
  - '2' : 点滅
- 例（緑1点灯、他全消灯）: [PC]:LED:10000000\n
2) 状態通知要求（LEDまたはBTN）
- フォーマット: [PC]:STA:LED\n  または  [PC]:STA:BTN\n
- 受信側（PI）は要求を受け取り次第、対応する状態通知を直ちに送信する。

PICO2 -> パソコン（通知）
1) LED 状態通知
- フォーマット: [PI]:LED:<8chars>\n
- 8chars 定義:
  - '0' : 消灯
  - '1' : 点灯
  - '2' : 点滅
- 例（生データ）: [PI]:LED:01210210\n
2) Button 状態通知（スナップショット）
- フォーマット: [PI]:BTN:<10chars>\n
- 10chars 定義:
  - '0' : OFF（押されていない）
  - '1' : ON（押されている）
- 例: [PI]:BTN:0100000000\n
3) Button 状態変化通知（イベント通知）
- フォーマット: [PI]:NTF:BTNx:<STATE>\n
- x: 0〜9（ボタン番号）
- STATE: OFF または ON（英字）
- 例: [PI]:NTF:BTN0:OFF\n  （ボタン0が離された）

エラー／ACK
- 正常受信確認（推奨）: PICO2 はコマンド受理時に ACK を返せる。
  - 形式: [PI]:ACK:<CMD>\n  （例: [PI]:ACK:LED）
- エラー応答: [PI]:ERR:<code>:<short_msg>\n
  - 例: [PI]:ERR:01:BAD_PAYLOAD （ペイロード長不一致）
- タイムアウト/リトライ方針は実装側で定義すること（推奨: 1秒内にACKが無い場合1回再送）

実装メモ / 注意点
- パーサは受信行を trim せず、正確な区切りと長さを検証すること。
- 余分な空白や改行混在を許可しないことで実装の単純化を図る。
- 将来拡張のため、タグ（[PC] / [PI]）やコマンド名は大文字固定とする。

デバッグ用の生データ例
- PC→PI: LED 全消灯:  "[PC]:LED:00000000\n"
- PC→PI: LED 指定なし（2番目だけ点滅）: "[PC]:LED:-2-0----\n"
- PC→PI: STA LED 要求: "[PC]:STA:LED\n"
- PI→PC: LED 状態通知: "[PI]:LED:11002200\n"
- PI→PC: BTN snapshot: "[PI]:BTN:0100000000\n"
- PI→PC: BTN event: "[PI]:NTF:BTN9:ON\n"
